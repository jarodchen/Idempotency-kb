# 使用指南

## 📖 如何使用本知识库

### 新手入门路径

如果你是第一次接触幂等性设计，建议按以下顺序学习：

#### 第一阶段：理论基础（1-2 天）

1. **理解核心概念**
   - 阅读 [`01-基础概念/1-什么是幂等性.md`](./01-基础概念/1-什么是幂等性.md)
   - 理解幂等性的定义和数学表达
   - 掌握幂等性与安全性的区别

2. **HTTP 方法与幂等性**
   - 阅读 [`01-基础概念/2-HTTP方法与幂等性.md`](./01-基础概念/2-HTTP方法与幂等性.md)
   - 了解哪些 HTTP 方法天然幂等
   - 学会如何为非幂等方法添加幂等性

3. **实际应用场景**
   - 阅读 [`01-基础概念/`](./01-基础概念/) 下的其他文档
   - 理解为什么需要幂等性
   - 识别项目中需要幂等性的场景

#### 第二阶段：实现方案（2-3 天）

4. **Token 机制**
   - 阅读 [`02-核心实现模式/2-Token机制-预生成模式.md`](./02-核心实现模式/2-Token机制-预生成模式.md)
   - 学习基于 Redis 的实现
   - 学习基于 PostgreSQL 的实现

5. **唯一索引**
   - 阅读 [`02-核心实现模式/4-唯一索引-数据库约束.md`](./02-核心实现模式/4-唯一索引-数据库约束.md)
   - 掌握数据库层面的幂等性保证
   - 学会处理唯一约束冲突

6. **其他实现模式**
   - 浏览 `02-核心实现模式` 目录下的其他文档
   - 了解乐观锁、分布式锁等方案

#### 第三阶段：技术实践（3-5 天）

7. **ASP.NET Core 实现**
   - 阅读 [`03-技术栈实现指南/1-NET-ASPNETCore中间件.md`](./03-技术栈实现指南/1-NET-ASPNETCore中间件.md)
   - 在你的项目中实现幂等性中间件
   - 编写单元测试

8. **数据库层实现**
   - 根据你的数据库类型选择对应的文档
   - PostgreSQL: 查看相关章节
   - MySQL: 查看相关章节

9. **Redis 实现**
   - 学习 Lua 脚本原子操作
   - 实现高性能的幂等性检查

#### 第四阶段：场景应用（2-3 天）

10. **支付系统**
    - 阅读 [`04-场景化解决方案/1-支付交易-防止重复扣款.md`](./04-场景化解决方案/1-支付交易-防止重复扣款.md)
    - 设计你的支付流程
    - 实现幂等性控制

11. **订单系统**
    - 阅读相关订单场景文档
    - 防止重复下单
    - 库存扣减幂等

12. **其他业务场景**
    - 根据你的业务需求查阅对应文档
    - 设计并实现幂等性方案

#### 第五阶段：优化与运维（持续）

13. **性能优化**
    - 阅读 [`06-常见问题与陷阱/1-性能-数据库锁竞争.md`](./06-常见问题与陷阱/1-性能-数据库锁竞争.md)
    - 监控幂等性系统的性能
    - 优化瓶颈

14. **监控告警**
    - 实现指标收集
    - 设置告警规则
    - 定期review

## 🎯 实战演练

### 练习 1：为现有 API 添加幂等性

**场景**：你有一个订单创建 API，需要添加幂等性支持。

**步骤**：

1. 阅读 `03-技术栈实现指南/1-NET-ASPNETCore中间件.md`
2. 在项目中添加幂等性中间件
3. 修改客户端代码，添加 `Idempotency-Key` 请求头
4. 编写测试验证幂等性

**预期结果**：
- 相同幂等键的请求只处理一次
- 后续请求返回缓存结果

### 练习 2：设计支付系统的幂等性

**场景**：设计一个支持微信、支付宝的支付系统。

**步骤**：

1. 阅读 `04-场景化解决方案/1-支付交易-防止重复扣款.md`
2. 设计数据库表结构
3. 实现支付服务
4. 处理第三方支付回调
5. 实现重试机制

**预期结果**：
- 不会重复扣款
- 支付回调幂等处理
- 网络异常可安全重试

### 练习 3：优化高并发场景的性能

**场景**：秒杀活动导致大量并发请求，数据库锁竞争激烈。

**步骤**：

1. 阅读 `06-常见问题与陷阱/1-性能-数据库锁竞争.md`
2. 引入 Redis 分布式锁
3. 实现异步队列削峰
4. 添加监控和告警

**预期结果**：
- 响应时间从 200ms 降低到 10ms
- 支持 10000+ QPS
- 无死锁和超时

## 💡 学习建议

### 1. 理论结合实践
- 每学完一个概念，立即动手实现
- 在你的实际项目中应用所学知识
- 从简单的场景开始，逐步复杂化

### 2. 深度优先，广度随后
- 先深入掌握一种实现方案
- 再横向对比其他方案
- 理解各方案的适用场景

### 3. 重视最佳实践
- 遵循业界成熟的设计模式
- 避免常见的陷阱和错误
- 关注性能和安全

### 4. 持续学习
- 关注新技术和最佳实践的演进
- 参与开源项目，学习他人经验
- 分享你的实践经验
